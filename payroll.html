Pay & Bill Adjustments

An internal payroll tool for EmployBridge.

Overview

EmployBridge is a staffing company that operates by paying workers placed on job assignments for billable clients. To ensure we're billing and paying accurately, a subset of colleagues is responsible for fixing pay & bill errors that occur outside of the usual payroll cycle, due to reasons such as late payroll changes or a client not approving hours before a deadline.

It was uncovered during earlier payroll-related research that the current adjustment process is inefficient and tedious for our colleagues. The goal of this work is to improve that process.

My Role

I was the sole designer on this project - responsible for initial ideation, iterating on designs, running usability testing, meeting with stakeholders, wireframing, prototyping and handing off designs. I collaborated with a product owner, business stakeholders, UX research, end users, and engineers throughout this process.

The Problem

Currently, colleagues create adjustments by filling out spreadsheets. This requires them to manually enter a lot of data and visit multiple databases outside of the spreadsheet to find necessary information. This workflow requires a lot of steps and is a pain point for our colleagues.

Design Process

The product owner put together a flow diagram that outlined the requirements for this project. After reviewing that, other discovery work, and relevant research, I started ideating initial wireframes.

I quickly realized I didn't fully understand aspects of the payroll workflow which made it hard to make certain design decisions, so I initiated feedback sessions with SMEs to help improve my wireframes. After a few rounds of iterations with input from them, I built out two prototypes for usability testing - one for each user group (adjustment submitter + adjustment approver). With guidance from the UX research team, I created a research brief, scripts & led 4 usability testing sessions to help validate the designs.

Minimizing risk of error

We eventually want to automate some adjustments rather than having them go through the approval process. Because of this, it's important to make sure adjustments are getting submitted correctly. I added in a confirmation modal to let a submitter review the adjustment and configured the adjustment page so a user can only edit fields related to the adjustment type selected (e.g. they can't edit hours if they're making a rate change).

Setting up for a future state

The focus of this first release is the main use case (standard rate and hour changes). However, we'll eventually build this tool out for other adjustment types. Even though those aren't fully supported on this first release, there's a way to submit documentation (i.e. a spreadsheet) for those other adjustments, as a way to encourage users to start adopting this system.

Results

This tool is currently in production and is expected to be released later this year. While we don't have metrics to share yet, all of the users from usability testing were excited about this tool which is encouraging. To learn more about this project or see a full prototype, contact me below.
